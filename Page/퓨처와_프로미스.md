[컴퓨터 과학에서](../Page/컴퓨터_과학.md "wikilink") **퓨처**(future), **프로미스**(promise), **딜레이**(delay), **디퍼드**(deferred)는 일부 [병행 프로그래밍 언어에서](../Page/병행_컴퓨팅.md "wikilink") 프로그램 [실행](https://ko.wikipedia.org/wiki/실행 "wikilink")을 동기화하기 위해 사용되는 구조체를 의미한다. 구조체를 의미한다. 프록시 역할을 하는 객체로 설명되며 값의 연산이 아직 이루어지지 않은 상태이므로 결과는 미리 알 수 없다.

프로미스(promise)라는 용어는 1976년 [대니얼 P. 프리드먼과](https://ko.wikipedia.org/wiki/대니얼_P._프리드먼 "wikilink") 데이비드 와이즈가 제안하였으며,\[1\] 피터 히버드는 eventual로 명명하였다.\[2\] 일부 비슷한 개념인 퓨처(future)는 1977년 [헨리 베이커와](https://ko.wikipedia.org/wiki/헨리_베이커 "wikilink") [칼 히위트의](https://ko.wikipedia.org/wiki/칼_히위트 "wikilink") 논문에 등장하였다.\[3\]

퓨처, 프로미스, 딜레이, 디퍼드라는 용어는 종종 번갈아 사용되지만 퓨처와 프로미스 사이에는 이용 면에서 일부 차이가 있다. 퓨처는 변수의 읽기 전용 플레이스홀더인 반면 프로미스는 미래의 값을 저장하는, 쓰기가 가능한 [단일 할당](https://ko.wikipedia.org/wiki/단일_할당 "wikilink") 컨테이너이다.\[4\] 퓨처는 어느 특정 프로미스가 값을 설정할지를 지정하지 않고 정의할 수 있으며 각기 다른 잠재적인 프로미스들이 주어진 퓨처의 값을 설정할 수 있지만 주어진 퓨처에 대해 오직 한 번만 수행이 가능하다. 그 밖의 경우 퓨처와 프로미스는 함께 만들어지며 서로 연결된다. 즉, 퓨처는 값이고 프로미스는 값을 설정하는 함수, 특히 비동기 함수(프로미스)의 반환값(퓨처)을 의미한다. 퓨처의 값을 설정하는 것은 바인딩(binding), 풀필링(fulfilling), 리졸빙(resolving)으로 부른다.

## 구현체 목록

일부 프로그래밍 언어들은 퓨처, 프로미스, 병행 논리 변수, 데이터플로 변수, l-var을 지원하며 이는 직접 언어 지원을 통해서나 표준 라이브러리를 통해 제공된다.

  - For [Groovy](../Page/그루비_\(프로그래밍_언어\).md "wikilink"): GPars\[30\]

      - [node](../Page/Node.js.md "wikilink")-promise\[37\]

      - Bluebird, by Petka Antonov\[42\]
      - The [Closure Library](../Page/구글_클로저_툴스.md "wikilink")'s [promise](https://github.com/google/closure-library/tree/master/closure/goog/promise) package conforms to the Promises/A+ specification.
      - See [Promise/A+'s](http://promisesaplus.com/implementations) list for more implementations based on the Promise/A+ design.
  - For [Java](../Page/자바_\(프로그래밍_언어\).md "wikilink"):
      - JDeferred, provides deferred-promise API and behavior similar to [JQuery](../Page/JQuery.md "wikilink").Deferred object\[43\]
      - ParSeq\[44\] provides task-promise API ideal for asynchronous pipelining and branching, maintained by [링크드인](../Page/링크드인.md "wikilink")
  - For [Lua](../Page/루아_\(프로그래밍_언어\).md "wikilink"):
      - The cqueues [1](https://www.25thandclement.com/~william/projects/cqueues.html) module contains a Promise API.
  - For [오브젝티브-C](../Page/오브젝티브-C.md "wikilink"): MAFuture,\[45\]\[46\] RXPromise,\[47\] ObjC-CollapsingFutures,\[48\] PromiseKit,\[49\] objc-promise,\[50\] OAPromise,\[51\]
  - For [OCaml](../Page/OCaml.md "wikilink"): Lazy module implements lazy explicit futures\[52\]
  - For [펄](../Page/펄.md "wikilink"): Future,\[53\] Promises,\[54\] Reflex,\[55\] and Promise::ES6\[56\]
  - For [PHP](../Page/PHP.md "wikilink"): React/Promise\[57\]
  - For [Python](../Page/파이썬.md "wikilink"):
      - Built-in implementation\[58\]
      - pythonfutures\[59\]
      - [Twisted's](../Page/트위스티드_\(소프트웨어\).md "wikilink") Deferreds\[60\]
  - For [R](../Page/R_\(프로그래밍_언어\).md "wikilink"):
      - future, implements an extendable future API with lazy and eager synchronous and (multicore or distributed) asynchronous futures\[61\]\[62\]
  - For [Ruby](../Page/루비_\(프로그래밍_언어\).md "wikilink"):
      - Promise gem\[63\]
      - libuv gem, implements promises\[64\]
      - Celluloid gem, implements futures\[65\]
      - future-resource\[66\]
  - For [Rust](../Page/러스트_\(프로그래밍_언어\).md "wikilink"):
      - futures-rs\[67\]
  - For [Scala](../Page/스칼라_\(프로그래밍_언어\).md "wikilink"):
      - Twitter's util library\[68\]
  - For [Swift](../Page/스위프트_\(프로그래밍_언어\).md "wikilink"):
      - Async framework, implements C\#-style `async`/non-blocking `await`\[69\]
      - FutureKit,\[70\] implements a version for Apple GCD\[71\]
      - FutureLib, pure Swift 2 library implementing Scala-style futures and promises with TPL-style cancellation\[72\]
      - Deferred, pure Swift library inspired by OCaml's Deferred\[73\]
      - BrightFutures\[74\]
  - For [Tcl](../Page/Tcl.md "wikilink"): tcl-promise\[75\]
