[섬네일](https://ko.wikipedia.org/wiki/파일:Linux_kernel_and_OpenGL_video_games.svg "wikilink") [섬네일](https://ko.wikipedia.org/wiki/파일:SDL_Layers.svg "wikilink") **심플 다이렉트미디어 레이어**(Simple DirectMedia Layer), **SDL**는 [C 프로그래밍 언어로](https://ko.wikipedia.org/wiki/C_프로그래밍_언어 "wikilink") 짜여진 [크로스플랫폼](https://ko.wikipedia.org/wiki/크로스플랫폼 "wikilink") 멀티미디어 라이브러리이다. 이 라이브러리는 비디오, 오디오, 사용자 입력 등의 계층을 [추상화하여](../Page/추상화_\(컴퓨터_과학\).md "wikilink"), [리눅스](../Page/리눅스.md "wikilink"), [마이크로소프트 윈도](https://ko.wikipedia.org/wiki/마이크로소프트_윈도 "wikilink"), [맥 OS X](https://ko.wikipedia.org/wiki/맥_OS_X "wikilink") 등 여러 [운영 체제에서](../Page/운영_체제.md "wikilink") 실행이 가능하도록 한다. 이 라이브러리는 비디오, 이벤트, 디지털 오디오, CD-ROM, 사운드, 스레드, 공유 객체 불러오기, 네트워킹, 타이머를 관리한다.\[1\]

이 라이브러리는 내부적으로 [C로](../Page/C_\(프로그래밍_언어\).md "wikilink") 작성되어 있으며, 대상 플랫폼에 따라 [C++](https://ko.wikipedia.org/wiki/C++ "wikilink")나 [오브젝티브-C](../Page/오브젝티브-C.md "wikilink")도 지원하며 C의 [API](../Page/API.md "wikilink")를 제공함으로써 다른 언어와의 결합을 지원한다.\[2\]

## 구조

SDL 라이브러리 그 자체는 매우 단순하다. 이 라이브러리에는 몇 가지 하부 시스템으로 나뉘는데, 이를테면 비디오, 오디오, CD-ROM, 조이스틱, 타이머가 있다. 이러한 낮은 수준의 기본적인 지원뿐 아니라 다음과 같은 기능들도 몇 가지 더 지원한다. 이들은 "표준 라이브러리"를 이루고, 공식 웹사이트에서 제공되며 공식 문서에 포함되어 있다:

  - *SDL_image* — 다수의 이미지 포맷 지원
  - *SDL_mixer* — 복잡한 오디오 함수 (주로 사운드 믹싱을 위해 존재)
  - *SDL_net* — 네트워킹 지원
  - *SDL_ttf* — [트루타입](../Page/트루타입.md "wikilink") 글꼴 렌더링 지원
  - *SDL_rtf* — 단순 [서식 있는 텍스트 포맷](../Page/서식_있는_텍스트_포맷.md "wikilink") 렌더링

### 언어 바인딩

SDL 2.0 라이브러는 [C](../Page/C_\(프로그래밍_언어\).md "wikilink"), [C++](https://ko.wikipedia.org/wiki/C++ "wikilink"), [파스칼](../Page/파스칼.md "wikilink")\[3\], [펄](../Page/펄.md "wikilink")(sdl.perl.org를 통해)\[4\], [파이썬](../Page/파이썬.md "wikilink")(PySDL2.0을 통해)\[5\], C\#\[6\], 루아\[7\], [OCaml](../Page/OCaml.md "wikilink")\[8\], [러스트](../Page/러스트_\(프로그래밍_언어\).md "wikilink"), [Nim](https://ko.wikipedia.org/wiki/Nim "wikilink"), [발라](../Page/발라_\(프로그래밍_언어\).md "wikilink"), [지니를](https://ko.wikipedia.org/wiki/지니_\(프로그래밍_언어\) "wikilink") 위한 [언어 바인딩이](https://ko.wikipedia.org/wiki/언어_바인딩 "wikilink") 있다.

## 예제

``` c
#include "SDL/SDL.h"

int main(int argc, char *args[])
{
    // SDL 비디오 시스템 초기화
    if (SDL_Init(SDL_INIT_VIDEO) == -1)
    {
        return 1;
    }

    // 화면 구성
    SDL_Surface *screen = NULL;
    screen = SDL_SetVideoMode(800, 600, 32, SDL_SWSURFACE);

    // 화면 구성에 오류가 있으면
    if (screen == NULL)
    {
        return 1;
    }

    // 창 위치 설정
    SDL_WM_SetCaption("간단 SDL 응용 프로그램!", NULL);

    // 프로그램의 종료 대기
    bool quit = false;

    // 프로그램이 아직 종료 상황이 아닌 동안
    while (quit == false)
    {
        // 이벤트가 있는 동안
        while (SDL_PollEvent(&event))
        {
            // 사용자가 창의 X 단추를 눌러서 프로그램을 끝낼 때
            if (event.type == SDL_QUIT)
            {
                // 프로그램 끝내기
                quit = true;
            }
        }

        // 화면 채우기
        SDL_FillRect(screen, &screen->clip_rect, SDL_MapRGB(screen->format, 0xFF, 0xFF, 0xFF));

        // 화면 갱신
        if (SDL_Flip(screen) == -1)
        {
            return 1;
        }
    }

    // SDL 끝내기
    SDL_Quit();

    return 0;
// 위 예제는 SDL2.0기반으로 작업시 오류가 발생함
}
```

## 역사

[섬네일](https://ko.wikipedia.org/wiki/파일:SDL-Workshop-IMG_2612.jpg "wikilink") 1998년 초에 처음 출시된 이 라이브러리는 [샘 랜틴가](https://ko.wikipedia.org/wiki/샘_랜틴가 "wikilink")(Sam Lantinga)가 [로키 소프트웨어용으로](https://ko.wikipedia.org/wiki/로키_소프트웨어 "wikilink") 작업하는 동안 만들어냈다. 그는 윈도 응용 프로그램을 매킨토시로 포팅하는 동안 이 개념을 떠올렸다. 그 뒤 SDL을 이용하여 [둠을](../Page/둠_\(1993년_비디오_게임\).md "wikilink") [BeOS](../Page/BeOS.md "wikilink")로 포팅하였다. 일부 다른 자유 라이브러리들이 SDL에 맞추어 개발되었는데, 이를테면 [SMPEG](https://ko.wikipedia.org/wiki/SMPEG "wikilink")와 [OpenAL](../Page/OpenAL.md "wikilink")이 있다. 그는 2008년에 갤럭시 게임웍스를 설립하여 SDL의 상업적인 지원을 돕기도 했으나 이 기업 계획은 시간 제한으로 인해 보류된 상태이다.\[9\]

SDL 1.3은 SDL 1.2 코드 기반의 주요 업데이트로 자리잡았다. 1.2 API의 여러 부분을, 다중 입출력 옵션을 위한 더 일반적인 지원으로 대체하고 있다. 추가되는 기능으로는 이를테면 다중 창 지원, 다중 입력 장치 지원 (다중 마우스, 다중 키보드 등), 하드웨어 가속 2차원 그래픽, 더 나은 [유니코드](../Page/유니코드.md "wikilink") 지원이 있다. 그 뒤 1.3/2.0으로 브랜치가 이동됨에 따라\[10\] SDL 1.3 개발이 끝나면 끝으로 SDL 2.0이 될 예정이다.\[11\] SDL 1.3은 [zlib 라이선스이므로](https://ko.wikipedia.org/wiki/zlib_라이선스 "wikilink") SDL 1.2와 달리 상용 클로즈드 소스 프로젝트에 정적 링크하여 자유롭게 이용할 수 있다.\[12\]
