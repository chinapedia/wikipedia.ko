**애들러-32**()는 1995년 [마크 애들러가](https://ko.wikipedia.org/wiki/마크_애들러 "wikilink") 발명한 [체크섬](../Page/체크섬.md "wikilink") [알고리즘](../Page/알고리즘.md "wikilink")으로서\[1\] [플레처 체크섬의](https://ko.wikipedia.org/wiki/플레처의_체크섬 "wikilink") 수정판이다. 동일한 길이의 [순환 중복 검사와](https://ko.wikipedia.org/wiki/순환_중복_검사 "wikilink") 비교하면, 속도에 대해 신뢰성을 타협한다. (즉, 속도를 더 우선한다) 애들러-32는 [플레처-16](https://ko.wikipedia.org/wiki/플레처-16 "wikilink")보다 더 미더우며, [플레처-32](https://ko.wikipedia.org/wiki/플레처-32 "wikilink")보다는 다소 덜 신뢰할만하다.\[2\]

## 역사

애들러-32 체크섬은 널리 쓰이는 [zlib](https://ko.wikipedia.org/wiki/zlib "wikilink") 압축 라이브러리의 일부이며, 이 또한 [마크 애들러가](https://ko.wikipedia.org/wiki/마크_애들러 "wikilink") 개발한 것이다. [롤링 체크섬](https://ko.wikipedia.org/wiki/롤링_체크섬 "wikilink") 버전의 애들러-32는 [rsync](https://ko.wikipedia.org/wiki/rsync "wikilink") 유틸리티에 사용된다.

## 예

[ASCII](https://ko.wikipedia.org/wiki/ASCII "wikilink") 문자열 "`Wikipedia`"의 애들러-32의 합은 다음과 같이 계산된다:

| 문자                        | ASCII 코드 |  | A     |       | B   |
| ------------------------- | -------- |  | ----- | ----- | --- |
| <small>(10진으로 표기)</small> |          |  |       |       |     |
| W                         | 87       |  | 1 +   | 87 =  | 88  |
| i                         | 105      |  | 88 +  | 105 = | 193 |
| k                         | 107      |  | 193 + | 107 = | 300 |
| i                         | 105      |  | 300 + | 105 = | 405 |
| p                         | 112      |  | 405 + | 112 = | 517 |
| e                         | 101      |  | 517 + | 101 = | 618 |
| d                         | 100      |  | 618 + | 100 = | 718 |
| i                         | 105      |  | 718 + | 105 = | 823 |
| a                         | 97       |  | 823 + | 97 =  | 920 |

`A =  920 =  0x398  (base 16)`
`B = 4582 = 0x11E6`
`출력 = 0x11E6 << 16 + 0x398 = 0x11E60398`
