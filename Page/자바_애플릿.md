**자바 애플릿**(Java applet)은 [자바 바이트코드](../Page/자바_바이트코드.md "wikilink") 형태로 배포되는 [애플릿](../Page/애플릿.md "wikilink")이다. [자바 가상 머신이](../Page/자바_가상_머신.md "wikilink") 내장된 [웹 브라우저나](../Page/웹_브라우저.md "wikilink") [썬 마이크로시스템즈에서](../Page/썬_마이크로시스템즈.md "wikilink") 배포하는 AppletViewer로 실행할 수 있다. 그 중 악의적인 툴인 자바애플릿 킬러는 모든 윈도 운영 체제를 공격할 수 있다. 자바 애플릿은 자바 언어의 첫 버전에 도입되었으며, 1995년 출시되었다.

자바 애플릿은 보통 자바로 작성되지만, [자이썬](../Page/자이썬.md "wikilink")\[1\], [JRuby](https://ko.wikipedia.org/wiki/JRuby "wikilink")\[2\] [파스칼](../Page/파스칼_\(프로그래밍_언어\).md "wikilink"),\[3\], [파스칼](../Page/파스칼_\(프로그래밍_언어\).md "wikilink"), [스칼라](../Page/스칼라_\(프로그래밍_언어\).md "wikilink"), [에펠](../Page/에펠_\(프로그래밍_언어\).md "wikilink")([스마트에펠](https://ko.wikipedia.org/wiki/스마트에펠 "wikilink")을 통해)\[4\]과 같은 다른 언어들 또한 사용할 수 있다.

자바 애플릿은 매우 빠른 속도로 수행되며, 2011년까지는 [자바스크립트](../Page/자바스크립트.md "wikilink") 보다 몇 배는 더 빨랐다.\[5\] [자바스크립트](../Page/자바스크립트.md "wikilink")와는 달리 자바 애플릿은 3차원 [하드웨어 가속](https://ko.wikipedia.org/wiki/하드웨어_가속 "wikilink") 접근이 가능했으므로 사소하지 않은, 연산 집중 시각 기능에 최적화시킬 수 있었다.

자바의 [바이트코드](../Page/바이트코드.md "wikilink")가 [크로스플랫폼](https://ko.wikipedia.org/wiki/크로스플랫폼 "wikilink")(플랫폼 독립적)이기 때문에 자바 애플릿은 [마이크로소프트 윈도우](../Page/마이크로소프트_윈도우.md "wikilink"), [FreeBSD](../Page/FreeBSD.md "wikilink"), [유닉스](../Page/유닉스.md "wikilink"), [macOS](https://ko.wikipedia.org/wiki/macOS "wikilink"), [리눅스](../Page/리눅스.md "wikilink")를 포함한 수많은 운영 체제용 브라우저(또는 기타 [클라이언트](https://ko.wikipedia.org/wiki/클라이언트 "wikilink"))를 통해 실행할 수 있다.

## 예제

다음 예제는 java.applet 패키지를 통해 자바 애플릿을 사용한 것을 나타낸 것이다. 이 예제는 자바 [애브스트랙트 윈도 툴킷](../Page/애브스트랙트_윈도_툴킷.md "wikilink")(AWT)의 클래스를 사용하여 [헬로 월드](https://ko.wikipedia.org/wiki/헬로_월드 "wikilink") 메시지를 출력한다.

``` java
import java.applet.*;
import java.awt.*;

// Applet code for the "Hello, world!" example.
// This should be saved in a file named as "HelloWorld.java".
public class HelloWorld extends Applet {

    // Print a message on the screen (x=20, y=10).
    public void paint(Graphics g) {
        g.drawString("Hello, world!", 20, 10);

        // Draws a circle on the screen (x=40, y=30).
        g.drawArc(40, 30, 20, 20, 0, 360);

      // Draws a rectangle on the screen (x1=100, y1=100, x2=300,y2=300).
        g.drawRect(100, 100, 300, 300);

      // Draws a square on the screen (x1=100, y1=100, x2=200,y2=200).
        g.drawRect(100, 100, 200, 200);



    }
}
```

단순한 애플릿들은 [플러그인](../Page/플러그인.md "wikilink")을 지원하는 응용 프로그램들을 정의하기 위해 인터넷에서 자유로이 공유된다.\[6\]

[컴파일](../Page/컴파일러.md "wikilink") 후의 결과물인 `.class` 파일이 웹 서버에 위치할 수 있으며 <applet>나 <object> 태그를 사용하여 [HTML](../Page/HTML.md "wikilink") 페이지 안에서 호출할 수 있다. 예는 다음과 같다:

``` html5
<!DOCTYPE html>
<html>
<head>
  <title>HelloWorld_example.html</title>
</head>
<body>
  <h1>A Java applet example</h1>
  <p>Here it is: <applet code="HelloWorld.class" height="40" width="200">
    This is where HelloWorld.class runs.
  </applet></p>
</body>
</html>
```

페이지에 접근할 때 다음과 같이 표시된다:

  -

      -
        <span style="font-size:1.9em;">**A Java applet example**</span>
        Here it is: Hello, world\!

다운로드 시간을 최소화하기 위해 애플릿은 [jar](../Page/JAR_\(파일_포맷\).md "wikilink") 파일 형태로 전달할 수 있다. 이 예제의 경우 모든 클래스들이 *example.jar*라는 압축된 형태 안에 위치할 경우 다음의 임베드되는 코드를 대신 사용할 수 있다:

``` html5
<p>Here it is: <applet archive="example.jar" code="HelloWorld" height="40" width="200">
  This is where HelloWorld.class runs.
</applet></p>
```

애플릿을 포함시키는 것은 APPLET 태그에 관한 썬의 공식 페이지에 상세 설명으로 기술되어 있다.\[7\]
