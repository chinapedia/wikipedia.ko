[소프트웨어 배포 생명 주기에](../Page/소프트웨어_배포_생명_주기.md "wikilink") 따라 [위키백과 모듈의](https://ko.wikipedia.org/wiki/위키백과:루아 "wikilink") 자체 평가를 위한 하나에서 다섯 가지의 분류:

  - [프리 알파](https://ko.wikipedia.org/wiki/:분류:개발_중인_프리_알파_모듈 "wikilink"): 미완성 모듈. 일반 이름공간 문서에서 사용할 수 없습니다. 원래 편집자 (또는 오랜 시간동안 중단 된 경우 대신 개발하는 사람)의 기본 구조에 만족 될 때까지 모듈은 프리 알파로 남아있게 됩니다.
  - [Alpha](https://ko.wikipedia.org/wiki/:분류:Modules_in_alpha "wikilink"): Modules which are ready for third party input. They may be used on a few pages to see if problems arise, but should be watched. Suggestions for new features or changes in their input and output mechanisms are welcome. Alpha modules should have /doc files describing the available parameters.
  - [Beta](https://ko.wikipedia.org/wiki/:분류:Modules_in_beta "wikilink"): Modules which are ready for more widespread use. They are still new and should be used with some caution to ensure the results are as expected. Ideally there should be a general announcement location for a module to enter beta status, and a community discussion to establish that it is ready for general use when it is time to release it, but there is not yet any specific procedure.
  - [Release](https://ko.wikipedia.org/wiki/:분류:Modules_for_general_use "wikilink"): Modules which reached a mature form and are thought to be bug-free and ready for use wherever appropriate. These are ready to mention on Help pages and other Wikipedia resources as an option for new users to learn. Modules at this level or above should be improved by sandbox testing rather than repeated trial-and-error editing to reduce the amount of reparsing and bad output.
  - [Protected](https://ko.wikipedia.org/wiki/:분류:Modules_subject_to_page_protection "wikilink"): Established modules in use by very large numbers of articles. Because vandalism or mistakes would affect so many pages, and even trivial editing would cause substantial load on the servers, these are subject to page protection.

The exact method by which modules or associated talk or documentation pages are to be tagged still needs some discussion...
