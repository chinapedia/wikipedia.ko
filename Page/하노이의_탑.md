[300px](https://ko.wikipedia.org/wiki/파일:Tower_of_Hanoi.jpeg "wikilink") **하노이의 탑**(Tower of Hanoi)은 퍼즐의 일종이다. 세 개의 기둥과 이 기둥에 꽂을 수 있는 크기가 다양한 원판들이 있고, 퍼즐을 시작하기 전에는 한 기둥에 원판들이 작은 것이 위에 있도록 순서대로 쌓여 있다.

게임의 목적은 다음 두 가지 조건을 만족시키면서, 한 기둥에 꽂힌 원판들을 그 순서 그대로 다른 기둥으로 옮겨서 다시 쌓는 것이다.

1.  한 번에 하나의 원판만 옮길 수 있다.
2.  큰 원판이 작은 원판 위에 있어서는 안 된다.

[300px](https://ko.wikipedia.org/wiki/파일:Tower_of_Hanoi_4.gif "wikilink") 하노이의 탑 문제는 [재귀 호출을](https://ko.wikipedia.org/wiki/재귀_호출 "wikilink") 이용하여 풀 수 있는 가장 유명한 예제 중의 하나이다. 그렇기 때문에 프로그래밍 수업에서 [알고리즘](../Page/알고리즘.md "wikilink") 예제로 많이 사용한다. 일반적으로 원판이 *n*개 일 때, 2<sup>*n*</sup> -1번의 이동으로 원판을 모두 옮길 수 있다(2<sup>*n*</sup> − 1는 [메르센 수라고](https://ko.wikipedia.org/wiki/메르센_수 "wikilink") 부른다).

참고로 64개의 원판을 옮기는 데 약 18446744073709551615번을 움직여야 하고, 한번 옮길 때 시간을 1초로 가정했을 때 64개의 원판을 옮기는 데 5849억 4241만 7355년이 걸린다.\[1\]

## 유래

하노이의 탑은 프랑스의 수학자인 [에두아르 뤼카](../Page/에두아르_뤼카.md "wikilink")(Édouard Lucas)가 클라우스 교수(professeur N. Claus)라는 필명으로 1883년에 발표하였다.\[2\] 1년 후 드 파르빌(Henri de Parville)은 Claus가 Lucas의 [애너그램](https://ko.wikipedia.org/wiki/애너그램 "wikilink")임을 밝히면서 다음과 같은 이야기로 하노이의 탑을 소개하였다.

  -
    [인도](../Page/인도.md "wikilink") [베나레스](https://ko.wikipedia.org/wiki/베나레스 "wikilink")에 있는 한 사원에는 세상의 중심을 나타내는 큰 [돔](https://ko.wikipedia.org/wiki/돔 "wikilink")이 있고 그 안에 세 개의 다이아몬드 바늘이 동판 위에 세워져 있습니다. 바늘의 높이는 1 [큐빗](https://ko.wikipedia.org/wiki/큐빗 "wikilink")이고 굵기는 벌의 몸통만 합니다. 바늘 가운데 하나에는 신이 64개의 순금 원판을 끼워 놓았습니다. 가장 큰 원판이 바닥에 놓여 있고, 나머지 원판들이 점점 작아지며 꼭대기까지 쌓아 있습니다. 이것은 신성한 [브라흐마](../Page/브라흐마.md "wikilink")의 탑입니다. 브라흐마의 지시에 따라 승려들은 모든 원판을 다른 바늘로 옮기기 위해 밤낮 없이 차례로 제단에 올라 규칙에 따라 원판을 하나씩 옮깁니다. 이 일이 끝날 때, 탑은 무너지고 세상은 종말을 맞이하게 됩니다.\[3\]

이후 [라우즈 볼](https://ko.wikipedia.org/wiki/라우즈_볼 "wikilink"), [가드너](../Page/마틴_가드너.md "wikilink") 등이 하노이의 탑을 소개하면서 널리 알려졌다.

## 소스 코드

이것은 각각의 수를 구하는 컴퓨터 프로그램 등의 실행어에 대한 것이다.

### [OCaml](../Page/OCaml.md "wikilink")

``` ocaml
let rec move_tower n a b c = match n with
    | 1 -> [(a,c)]
    | _ -> (move_tower (n-1) a c b) @ (move_tower 1 a b c) @ (move_tower (n-1) b a c);;
```

### [Lisp](../Page/리스프.md "wikilink")

``` lisp
(defun hanoitowers (disc src aux dst)
  (cond ((> disc 0)
     (hanoitowers (- disc 1) src dst aux)
     (princ (list "Move" disc "from" src "to" dst))
     (hanoitowers (- disc 1) aux src dst))))
```

### [Pascal](../Page/파스칼_\(프로그래밍_언어\).md "wikilink")

``` pascal
procedure Hanoi(n: integer; from, dest, by: char);
Begin
    if (n=1) then
        writeln('Move the plate from ', from, ' to ', dest)
    else begin
        Hanoi(n-1, from, by, dest);
        Hanoi(1, from, dest, by);
        Hanoi(n-1, by, dest, from);
    end;
End;
```

### [C](../Page/C_\(프로그래밍_언어\).md "wikilink")

``` cpp
#include <stdio.h>

void move(int from, int to)
{
    printf("\nMove from %d to %d", from, to);
}

void hanoi(int n, int from, int by, int to)
{
    if (n == 1)
        move(from, to);
    else
    {
        hanoi(n - 1, from, to, by);
        move(from, to);
        hanoi(n - 1, by, from, to);
    }
}
```

### [Python](../Page/파이썬.md "wikilink")

``` python
def hanoi(n, from_pos, to_pos, aux_pos):
    if n == 1:
        print(from_pos, "->", to_pos)
        return
    hanoi(n - 1, from_pos, aux_pos, to_pos)
    print(from_pos, "->", to_pos)
    hanoi(n - 1, aux_pos, to_pos, from_pos)
```
