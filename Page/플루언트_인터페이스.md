[소프트웨어 공학에서](../Page/소프트웨어_공학.md "wikilink") **플루언트 인터페이스**(fluent interface)는 [메소드 체이닝에](https://ko.wikipedia.org/wiki/메소드_체이닝 "wikilink") 상당 부분 기반한 [객체 지향](https://ko.wikipedia.org/wiki/객체_지향 "wikilink") API 설계 메소드이며, 소스 코드의 가독성을 [산문](https://ko.wikipedia.org/wiki/산문 "wikilink")과 유사하게 만드는 것이 목적이다. 특히 인터페이스 안에 [도메인 특화 언어를](../Page/도메인_특화_언어.md "wikilink") 작성한다. 에릭 에반스와 마틴 폴러가 처음 만든 용어이다. JMock 테스팅 프레임워크에서 유연한 테스트 예측의 예는 다음과 같다:\[1\]

``` Java
mock.expects(once()).method("m").with( or(stringContains("hello"),
                                          stringContains("howdy")) );
```

## 예시

### C\#

``` csharp
var translations = new Dictionary<string, string>
                   {
                       {"cat", "chat"},
                       {"dog", "chien"},
                       {"fish", "poisson"},
                       {"bird", "oiseau"}
                   };

// Find translations for English words containing the letter "a",
// sorted by length and displayed in uppercase
IEnumerable<string> query = translations
    .Where   (t => t.Key.Contains("a"))
    .OrderBy (t => t.Value.Length)
    .Select  (t => t.Value.ToUpper());

// The same query constructed progressively:
var filtered   = translations.Where (t => t.Key.Contains("a"));
var sorted     = filtered.OrderBy   (t => t.Value.Length);
var finalQuery = sorted.Select      (t => t.Value.ToUpper());
```
