위키백과에서 IP가 차단될 때, ‘익명 사용자만 차단’ 기능이 꺼진 경우 해당 IP를 이용하는 사용자들이 위키백과를 편집할 수 없게 됩니다. 이러한 차단이 이루어지는 이유는 해당 IP 뿐만 아니라 그 IP를 이용하여 로그인하는 사용자들의 편집까지 막아야 할 경우가 발생하기 때문입니다. 이 경우 위키백과를 차단하는 국가 등에서 공개 프록시를 사용해 접속하는 사용자나, 학교 등에서 다른 사람이 문서 훼손을 하여 차단된 경우 뜻하지 않게 편집을 못하는 상황이 발생할 수 있습니다. 이로 인한 선의의 피해자 발생을 방지하기 위하여 위키백과에는 **IP 차단 면제** 기능이 있습니다.

  - 이 옵션이 부여된 사용자는 차단된 IP 주소에서라도 로그인하여 문서를 편집할 수 있습니다.
  - 차단된 IP에서 로그인하여 편집한 경우 [검사관](https://ko.wikipedia.org/wiki/백:검사관 "wikilink")(CheckUser)이 확인할 수 있는 기록에 남습니다.
  - [관리자는](https://ko.wikipedia.org/wiki/백:관리자 "wikilink") IP 차단 면제자 권한을 명시적으로 가지고 있지 않지만, 그에 상응하는 기능을 포함하고 있습니다.
  - 이 권한은 관리자가 요청에 따라 부여하고 회수하는 것이 가능합니다.

## 플래그 요청

일반적으로 IP 차단 면제 플래그를 부여받기 위해서, 다음과 같은 사항을 관리자에게 확인하여 주어야 합니다.

  - 요청자는 어떤 IP 또는 IP 대역에서 차단이 이루어졌는지, 요청 시에 밝혀야 합니다.
      -
        해당 정보는 편집을 시도할 때 편집이 차단되었다면 나타나는 정보입니다. 이 정보를 제시하여야 하는 이유는 해당 IP나 IP 대역에서 실제로 로그인한 사용자가 차단되는지 차단 설정을 확인하여야 하기 때문입니다. 관리자는 IP 차단 면제 플래그를 부여하기보다 해당 IP나 IP 대역의 차단 설정을 바꿈으로써 문제를 해결할 수도 있습니다.
  - 요청자는 IP 차단 면제 플래그를 이용하여 악의적 목적으로 사용되어 차단된 프록시를 통한 편집을 하지 않을 것을 확인하여야 합니다.
      -
        만약 불가피하게 차단된 프록시를 사용하게 될 경우 관리자에게 어떤 프록시 우회 사이트나 어떤 주소를 사용하는지 반드시 밝혀주어야 합니다. 차단된 프록시를 사용하는 편집은 검사관(CheckUser)이 열람 가능한 기록에 남습니다.

이 플래그는 IP가 아닌 **해당 IP를 사용하는 계정**에 부여됩니다. 그러므로 반드시 계정을 갖고 있어야 요청할 수 있습니다. IP 차단 면제 플래그의 요청은 [사용자 권한 신청에서](https://ko.wikipedia.org/wiki/백:사용자_권한_신청 "wikilink") 요청할 수 있습니다.

## 플래그 회수

차단된 IP나 IP 대역의 차단 기한이 끝났거나 차단 설정이 바뀌는 경우, 해당 플래그는 회수될 수 있습니다. 또한 IP 차단 면제 플래그를 요청 당시의 목적 또는 요청의 내용과 달리 악용한 것이 밝혀질 경우 즉시 플래그는 회수됩니다. 또한 이러한 악용은 장기간 차단의 사유가 될 수 있습니다. 그리고 해당 플래그의 기능을 더 이상 이용할 필요가 없을 때에도 플래그는 회수될 수 있습니다.
